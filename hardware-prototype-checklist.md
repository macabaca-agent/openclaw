# 硬件原型协作清单

最后更新：2026-03-25

适用于：Arduino、ESP32、各类传感器、可穿戴原型、交互装置、智能布料/智能服装实验。

## 目标

当用户需要做硬件原型时，用统一方式整理信息、推进调试、减少来回沟通成本。

## 用户每次最好先提供的信息

### 1. 目标
- 这个原型要实现什么功能？
- 输入是什么？
- 输出是什么？
- 想做演示、实验验证，还是长期使用？

### 2. 硬件清单
- 主控型号（如 Arduino Uno、Nano、ESP32 等）
- 传感器型号
- 执行器型号（LED、马达、震动、加热、蜂鸣器等）
- 电源方式（USB / 电池 / 外接供电）
- 是否有面包板、模块、驱动板

### 3. 连接情况
- 目前怎么接线
- 哪些引脚接了什么
- 最好附照片或草图

### 4. 软件情况
- 当前代码
- 是否能编译
- 是否能上传
- 串口有没有输出
- 报错内容是什么

### 5. 当前卡点
- 完全没反应
- 数据不稳定
- 串口异常
- 供电问题
- 某个交互效果做不出来

## 我可以提供的帮助

### 设计与规划
- 拆解原型目标
- 设计交互逻辑
- 选传感器/执行器方案
- 设计状态机、数据流、反馈方式

### 编程与联动
- 写 Arduino / ESP32 代码
- 写串口通信脚本
- 写网页与硬件联动逻辑
- 帮助做可视化和演示流程

### 调试与排错
- 看报错
- 分析代码逻辑
- 推断接线问题
- 帮助定位供电、通信、时序问题

## 协作建议流程

1. 先说目标
2. 列硬件清单
3. 发接线情况
4. 发代码或报错
5. 先做最小可运行版本
6. 再逐步加复杂功能

## 对可穿戴/智能布料项目的额外提醒

- 优先注意供电安全
- 贴身穿戴时谨慎处理发热与短路风险
- 柔性材料、导电线、布料连接点容易出现接触不良
- 演示版与可长期使用版应分开考虑

## 高风险项目注意事项

以下情况不要默认直接操作，应先复核：
- 电池供电
- 发热元件
- 电机驱动
- 较大电流负载
- 贴身穿戴供电
- 自制电路板焊接后首次上电
