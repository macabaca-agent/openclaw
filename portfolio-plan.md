# 作品集网页规划

最后更新：2026-03-25

## 目标

制作一个同时面向企业招聘方与学术场景的个人作品集网页，既能展示研究能力，也能展示个人个性。

## 已确认方向

- 受众：企业 + 学术
- 风格：酷一点的实验性设计
- 体验：需要动效
- 技术：允许复杂，重视长期维护

## 网站核心目标

1. 清楚展示个人定位
2. 展示代表性项目与研究
3. 体现“智能布料 / 智能服装 / 交互设计”的独特背景
4. 支持后续长期维护与持续更新
5. 为后续记录“和小龙虾一起求职 / 一起做网页”的视频内容预留素材与叙事空间

## 建议的信息架构（第一版）

1. Home
   - 一句话自我定位
   - 视觉化首屏
   - 进入项目区入口

2. About
   - 背景介绍
   - 研究兴趣
   - 技能关键词
   - 当前求职状态

3. Selected Projects
   - 3–5 个重点项目卡片
   - 每个项目进入详情页

4. Research / Experiments
   - 更偏学术或实验性的内容
   - 论文、研究主题、实验原型、探索过程

5. CV
   - 简历摘要
   - 下载 PDF（后续）

6. Contact
   - 邮箱
   - GitHub / LinkedIn / 其他链接（后续）

## 每个项目建议结构

- 项目标题
- 时间
- 项目类型（研究 / 原型 / 设计 / 装置 / 论文相关）
- 背景与问题
- 目标
- 方法
- 过程
- 最终产出
- 你的角色与贡献
- 图片 / 视频 / 草图 / 原型演示
- 反思与延展

## 视觉与交互方向（初步）

- 整体气质：实验性、未来感、但不要难读
- 动效方向：
  - 首屏轻量动效
  - hover 反馈
  - 页面切换过渡
  - 项目图像/文本 reveal 动画
- 原则：效果服务内容，不做纯炫技

## 技术方向（初步建议）

优先考虑：Next.js

原因：
- 适合长期维护
- 便于做多页面和项目详情页
- 后续可扩展多语言、博客、研究记录
- 部署方便（如 Vercel）

可选补充：
- Tailwind CSS（加快样式搭建）
- Framer Motion（实现动效）
- MDX 或 JSON/TS 数据驱动项目内容
