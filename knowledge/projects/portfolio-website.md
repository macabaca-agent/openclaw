# Portfolio Website

## Goal
- Build a first version of a motion-focused portfolio website by the end of March 2026.
- It should work for both job applications and academic presentation.
- The site should feel personal, a bit experimental/cool, and have motion, while still being maintainable long-term.

## Current understanding
- Scope should stay small and shippable.
- Prioritized pages likely include: Home, Projects, Project Detail, About/Contact.
- Motion should be focused on a few high-value areas rather than everywhere.
- Main execution may happen on the user's primary machine with Claude Code; this workspace acts as support, planning, prompt design, review, and remote fixes.

## Design direction
- Wants personality, not a generic template.
- Prefers curated options over one rigid direction.
- Experimental/cool is good, but should not become messy or impossible to maintain.

## Build strategy
- Ship a strong first version first.
- Favor clear structure, maintainable components, and selective motion.
- Use AI efficiently: planning and prompt design here, main implementation on the primary machine.
